# 用户操作手册

iPhone 定位模拟控制器 · 网页版

---

## 一、程序启动

1. 双击项目根目录的 **`启动.bat`**（它调用 `launch.ps1`）。
   - 首次启动会自动创建环境并安装依赖（走清华镜像源，约 1–5 分钟），请耐心等待。
   - 脚本会**在打开前端网页时同步启动所有必要的后端程序**，无需手动另起后端。
2. 依赖装好后，浏览器会自动打开 `http://127.0.0.1:8765`。
3. 出现黑色界面、左侧控制面板 + 右侧世界地图即表示启动成功。
4. 保持**命令行黑窗口**开着（关闭它等于退出程序）。启动器窗口随后端运行保持打开，后端退出后自动关闭。

> 想不接真机先熟悉操作？命令行运行 `启动.bat dry` 进入模拟模式。
> 若 pymobiledevice3 装不上，脚本会自动回退 dry-run 模式，确保网页总能打开。
> 旧版 `run.bat` 仍可用，逻辑等价。

界面顶部两个徽章说明：
- **环境徽章**：「环境就绪」=依赖完整；「pymobiledevice3 未安装」=需补依赖；「模拟模式」=dry-run。
- **连接徽章**：显示当前设备连接状态。

## 二、连接 iPhone（真机模式）

> 首次使用前，请先按 README 完成 Python、iTunes/Apple Devices、iPhone 开发者模式的准备。

1. 用 USB 数据线把 iPhone 连到电脑，**解锁手机**。
2. 手机弹出「要信任此电脑吗？」→ 点「**信任**」→ 输入锁屏密码。
3. 在程序左侧「① 设备连接」卡片，点「**🔄**」刷新设备列表。
4. 下拉框选中你的 iPhone（显示设备名 + iOS 版本 + UDID 前 8 位）。
5. 点「**连接**」。成功后顶部徽章变绿「已连接: xxx」，地图可开始规划。

若连接失败，错误提示会直接告诉你原因（未信任 / 未开开发者模式 / 未装驱动等），按提示处理后重试。详见 README「常见问题排查」。

### 开启开发者模式（iOS 16+，菜单默认隐藏时）

iPhone 16 等 iOS 16+ 设备的「开发者模式」选项默认在「设置 → 隐私与安全性」中**看不到**，需先用本程序触发一次才会出现/启用。本程序内置一键触发：

1. 在「① 设备连接」点「🔄」刷新并选中你的 iPhone（此时下方「🔧 开启开发者模式」按钮变为可用，并自动查询当前状态）。
2. 点「**🔧 开启开发者模式**」。
   - 程序经 Apple 官方 `amfi` 通道**在手机上自动调出「启用开发者模式」窗口**。
   - 按手机提示：点「启用」→ 输入锁屏密码 → 确认**重启**。
   - 重启完成后回到网页点「连接」即可。
3. 若只想先让「开发者模式」选项出现在设置菜单里、再自行手动开，可点旁边的「**👁 显示菜单**」，然后去手机「设置 → 隐私与安全性」里手动打开并重启。

> 该功能调用 `pymobiledevice3 amfi enable-developer-mode / reveal-developer-mode`，经普通 USB lockdown 即可，无需隧道或管理员权限。dry-run 模式下该按钮同样可点（模拟成功），便于演示。

## 三、规划运动路径

支持三种添加航点方式，可混用：

| 方式 | 操作 |
|------|------|
| **地图点击** | 在地图任意位置单击，即添加一个航点（蓝色水滴，带序号） |
| **拖拽微调** | 鼠标按住已有航点拖动，可调整位置，路径实时重绘 |
| **坐标输入** | 在「纬度 / 经度」框填入数值，点「添加」（自动定位到该点） |
| **删除航点** | 在航点上**右键**即可删除该点 |
| **撤销** | 点「↩ 撤销」删除最后一个航点 |
| **清空** | 点「清空路径」删除全部 |

- 第 1 个航点为**绿色**（起点），最后一个为**红色**（终点），中间为蓝色。
- 航点用蓝色折线相连，即为运动轨迹。
- 「路径规划」卡片下方实时显示：**航点数 / 总距离(km) / 预计耗时**（按当前速度计算）。

**预览轨迹**：点「▶ 预览路径轨迹（不下发设备）」，绿色虚线 + 移动圆点会在地图上动画演示整条路径，**不会影响手机**，供你确认路线无误。预览时长自动压缩在 2–15 秒。

## 四、设置速度

- **滑块**：拖动「速度」滑块（0.1–200 km/h）。
- **数值输入**：右侧数字框可精确输入（0.1–2000 km/h，超出滑块范围时仍生效）。
- **快捷预设**：点下方圆角按钮一键设为 步行4 / 骑行15 / 驾车40 / 高速100 km/h。
- 速度变化后，总距离下方的「预计耗时」会即时重算。

## 五、开始模拟运动

> 前提：已连接设备 + 路径 ≥2 个航点 + 速度 >0。

「④ 模拟控制」卡片顶部有两个可选设置项，可在点「开始」前按需配置：

- **🔁 循环次数**（单位：次）
  - `0`（默认）= **单次执行**：跑完当前轨迹即停。
  - 正整数 `N` = **精确重复执行当前轨迹 N 次**：每跑完一遍回到起点继续下一遍，共跑 N 遍。
  - 例如填 `5`，则同一条路径会连续跑 5 圈。
- **👟 随机误差**（开关）
  - 打开后，模拟坐标会叠加 **±5 米以内**的自然跑步随机偏移：方向随机性 + 步长波动，使轨迹更接近真实人跑动（而非完美贴在折线上），有助于规避部分 App 的「直线异常」检测。
  - 关闭则严格沿规划路径行进。
  - 误差采用平滑随机游走算法，相邻点不会突变跳变，且**严格钳制在 ±5 米内**。

1. 点「**▶ 开始**」。
2. 地图上出现**红色实时定位点**，沿路径平滑移动；右侧「运行状态」面板同步显示：
   - 状态（运行中/已暂停/已完成…）
   - 当前坐标
   - 已行进 / 总距离 + 进度条
   - 当前速度
   - **循环进度**（当前第几圈 / 总圈数；单次执行时显示「单次」）
   - **随机误差**（开 ±5m / 关）
3. 控制按钮：
   - **⏸ 暂停**：停在当前位置不动（手机定位也停住）。
   - **⏵ 继续**：从暂停处继续前进。
   - **⏹ 停止**：结束本次模拟，停在当前位置（**不恢复真实定位**）。再次「开始」会从起点重跑。
4. 跑完全程（或全部循环）后状态变为「已完成」，定位停在终点。

> 循环与随机误差均与暂停/继续/停止无缝兼容：暂停会冻结当前进度（含当前圈次），继续后从断点接着跑，循环计数不受影响。

## 六、恢复实际定位（重要）

点「**🔄 恢复实际定位**」→ 确认对话框点「确认」：
- 立即停止任何进行中的模拟；
- 清除 iPhone 上的定位覆盖；
- 手机**即时回到真实 GPS**。
- 地图上的红色实时点消失。

> 建议在不需要模拟时随时点此按钮，让手机回到真实定位。

---

## 七、退出程序

点「**✕ 退出程序**」→ 确认：
- 自动恢复真实定位（清除覆盖）；
- 终止所有模拟线程与设备连接；
- 关闭后端服务。

页面会显示「程序已退出，可关闭此窗口」，随后可关闭浏览器与命令行窗口。
> 也可直接关闭命令行黑窗口强制退出（退出前不会自动恢复定位，建议优先用「退出程序」按钮）。

---

## 八、地图操作小贴士

- **切换图层**：右上角「🗺 街道 / 🛰 卫星」按钮切换街道图与卫星图。
- **缩放/平移**：滚轮缩放、拖动平移，与主流地图一致。
- **默认视野**：启动时默认显示北京，可随意拖到任意区域规划路径。

## 九、故障排除

| 现象 | 处理 |
|------|------|
| 点「开始」提示「请至少添加 2 个航点」 | 在地图上多点几个航点 |
| 点「开始」提示「尚未连接设备」 | 先按第二节连接设备 |
| 连接报「未开启开发者模式」 | 选中设备后点「🔧 开启开发者模式」，按提示在手机确认并重启；重启后回这里点「连接」 |
| iPhone 设置里找不到「开发者模式」选项 | iOS 16+ 默认隐藏。选中设备后点「👁 显示菜单」令选项出现，或直接点「🔧 开启开发者模式」走自动启用流程 |
| 模拟运行中手机定位不动 | ① 确认开发者模式已开；② 地图类 App 内重启定位 / 杀掉重开 |
| 模拟卡顿 | 关闭其他占 CPU 的程序；路径点不宜过密（>500 点可适当精简） |
| 浏览器标签页崩溃 |Leaflet 卫星图层在大范围缩放时较占内存，可切回街道图 |
| 启动器提示「后端启动失败并已退出」 | 多为端口 8765 被占用：关闭占用程序后重试，或手动 `python backend\server.py --port 8766` |
| 误关闭命令行窗口 | 重新双击 `启动.bat` 即可，模拟状态会重置 |

## 十、隐私说明

- 本程序不读取、不存储、不上传你的真实位置。
- 所有路径规划与模拟计算均在你的电脑本地完成。
- 「恢复实际定位」与「退出程序」会确保手机回到真实 GPS。
